# OpenClaw 车机安卓前端应用 开发说明书
## 文档版本：V1.0
## 适用场景：安卓系统车机、OpenClaw Agent 配套前端配置展示APP

---

## 一、项目概述
### 1.1 项目背景
本项目为**OpenClaw 智能体（Agent）** 配套的车机端安卓原生应用，核心作用是为用户提供简洁、明亮的可视化操作界面，完成Agent人设配置、音色设置、技能查看、用户记忆查看四大核心功能，实现与OpenClaw Agent的文件级数据交互。

### 1.2 核心定位
- 运行环境：**安卓系统车机**（横屏、触控操作，适配车机分辨率）
- 界面风格：**简洁、明亮、清晰，强制浅色模式，无暗黑/深色主题**
- 核心功能：人设/音色配置写入、技能/记忆读取展示
- 交互方式：标签页切换、点击选择、列表展示

### 1.3 技术栈
- 开发语言：Kotlin（安卓官方推荐，兼容Java）
- 开发框架：Android Jetpack、Material 3（明亮风格UI组件）
- 核心组件：ViewPager2 + TabLayout（标签页切换）、RecyclerView（列表展示）
- 数据交互：安卓本地文件读写（与OpenClaw Agent配置文件对接）

---

## 二、开发环境要求
### 2.1 开发工具
- Android Studio Hedgehog | 2023.1.1 及以上版本
- JDK 17 及以上版本

### 2.2 系统适配
- 最低支持：Android 8.0 (API 26)
- 目标支持：Android 10.0~13.0（主流车机安卓系统）
- 屏幕适配：**横屏固定**（禁止竖屏旋转），适配7寸/9寸/12.3寸车机中控屏

### 2.3 权限要求
应用需申请安卓文件读写权限，用于读取/写入OpenClaw Agent配置文件：
```xml
<!-- 清单文件权限配置 -->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.MANAGE_EXTERNAL_STORAGE" />
```

## 三、需求详细说明
### 3.1 UI设计需求（核心：简洁明亮，无深色模式）
1. **整体布局**
   - 页面顶部：**三个固定标签页**（人设、技能、记忆），标签文字清晰、间距适中
   - 页面主体：标签页对应的内容区域，白色背景，浅色系控件，圆角设计
   - 交互反馈：点击/选择有轻微高亮效果，无复杂动画

2. **禁止项**
   - 禁用系统暗黑模式、深色主题
   - 禁用高对比度、深色背景
   - 禁用复杂炫彩样式，保持极简清爽

### 3.2 功能需求
1. **人设标签页**
   - 内置3种默认身份：**实干派、贴心友人、全能派**，点击即可选中并写入配置文件
   - 音色设置栏：内置4种常用车机音色（标准男声、标准女声、温柔女声、沉稳男声），选择后写入语音模型配置文件
2. **技能标签页**
   - 从OpenClaw Agent指定文件中读取技能列表，纯展示，不可编辑
3. **记忆标签页**
   - 从OpenClaw Agent指定文件中读取用户记忆数据，纯展示，不可编辑

## 五、核心功能模块设计
### 5.1 文件路径约定（与OpenClaw Agent强绑定，固定不可修改）
为保证应用与Agent正常通信，**所有配置文件路径固定**，应用直接读写以下路径：
```
# 人设身份提示词文件（点击身份写入）
/sdcard/Android/data/com.openclaw.car/files/openclaw/agent/persona.txt

# 音色配置文件（选择音色写入）
/sdcard/Android/data/com.openclaw.car/files/openclaw/tts/voice_config.txt

# 技能数据文件（读取展示）
/sdcard/Android/data/com.openclaw.car/files/openclaw/agent/skills.txt

# 记忆数据文件（读取展示）
/sdcard/Android/data/com.openclaw.car/files/openclaw/agent/memory.txt
```

### 5.2 功能模块拆分
#### 模块1：标签页切换模块
- 基于ViewPager2 + TabLayout实现三个页面的无缝切换
- 切换无卡顿，适配车机触控操作

#### 模块2：人设配置模块（核心写入功能）
1. **内置身份预设提示词**（可根据需求修改）
   - 实干派：`你是一个务实高效的助手，专注解决问题，回答简洁直接，不冗余`
   - 贴心友人：`你是一个温暖贴心的朋友，语气亲切柔和，善于倾听和安慰`
   - 全能派：`你是一个全能助手，精通各类知识，能解答问题、提供建议、辅助决策`
2. **写入逻辑**
   - 用户点击身份卡片 → 应用将对应提示词**覆盖写入**`persona.txt`
   - 用户选择音色 → 应用将音色标识**覆盖写入**`voice_config.txt`
3. **状态保存**：记录用户最后一次选择的身份/音色，打开应用自动回显

#### 模块3：技能展示模块（读取功能）
- 应用启动/切换到技能页 → 读取`skills.txt` → 解析文本 → 列表展示
- 文件格式：纯文本/换行分割（每行一个技能）

#### 模块4：记忆展示模块（读取功能）
- 应用启动/切换到记忆页 → 读取`memory.txt` → 解析文本 → 滚动展示
- 文件格式：纯文本/换行分割（每条记忆一行）

## 六、核心代码实现（关键片段）
### 6.1 强制浅色主题（styles.xml）
```xml
<!-- 禁用深色模式，强制明亮风格 -->
<style name="Theme.OpenClawCar" parent="Theme.Material3.Light.NoActionBar">
    <item name="android:forceDarkAllowed">false</item>
    <item name="android:windowBackground">@color/white</item>
    <item name="colorPrimary">@color/light_blue</item>
</style>
```

### 6.2 人设身份写入逻辑（Kotlin）
```kotlin
// 点击实干派按钮
btn_shipai.setOnClickListener {
    val content = "你是一个务实高效的助手，专注解决问题，回答简洁直接，不冗余"
    writeFileToAgent(PERSONA_FILE_PATH, content)
    showToast("已设置为：实干派")
}

// 文件写入工具方法
private fun writeFileToAgent(path: String, content: String) {
    try {
        val file = File(path)
        file.parentFile?.mkdirs() // 创建文件夹
        file.writeText(content, Charsets.UTF_8)
    } catch (e: Exception) {
        e.printStackTrace()
        showToast("配置写入失败")
    }
}
```

### 6.3 技能/记忆读取逻辑
```kotlin
// 读取文件并展示
private fun readSkillFile(): List<String> {
    val file = File(SKILL_FILE_PATH)
    if (!file.exists()) return emptyList()
    return file.readLines(Charsets.UTF_8) // 按行读取技能
}
```

## 七、部署与安装
### 7.1 应用打包
- 使用Android Studio生成**release版APK**
- 包名：`com.openclaw.car`

### 7.2 车机部署方式
1. 车机开启「开发者选项」→ 允许USB调试
2. 电脑连接车机，使用ADB命令安装：
   ```bash
   adb install openclaw_car.apk
   ```
3. 车机桌面找到「OpenClaw助手」图标，打开应用

### 7.3 初始化配置
首次打开应用，需授权**文件读写权限**，否则无法读写Agent配置文件。

## 九、注意事项
1. **文件路径一致性**：应用与OpenClaw Agent必须使用相同的文件路径，否则无法通信
2. **车机权限**：安卓10+车机必须授予「所有文件访问权限」，否则无法读写文件
3. **UI适配**：车机屏幕较大，按钮/文字尺寸需放大，保证触控便捷
4. **数据安全**：仅读写指定配置文件，不修改其他系统文件
5. **稳定性**：应用后台保活，适配车机休眠/唤醒机制
